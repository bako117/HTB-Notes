### Preface



Automated Tools are frequently debated. Some argue it is too easy and ruins everything. Downsides of automated tooling includes: 

* Creating a comfort zone
* Creating risk by having tools be publicly available
* Creating tunnel vision mindset "If tool can't, I can't" 



HTB Argues for pentesters in a commercial environment: 

* Time=Money for everyone, especially the client
* High quality + consistency + quantity work is appreciated
* You only need to impress yourself, not the infosec community. By achieving what you want to achieve and pushing yourself, recognition will automatically present itself. 



### Metasploit



The Metasploit Project is Ruby-based penetration testing platform known for its modularity and ability to write, test, and execute exploit code. Code can be custom or from a database. 



At the highest level, it provides common tools and creates a complete environment for penetration testing and exploit development. The main folders withing the library for Metasploit are:


**Data, Documentation, Lib -** Technical details + functioning interface parts

**Modules -** 

**Plugins -** You can add plugins to Metasploit to add more functionality + automation. 

**Scripts -** Useful scripts + meterpreter (reverse shell) functionality

**Tools -** Command Line utilities that can called directly from msfconsole
